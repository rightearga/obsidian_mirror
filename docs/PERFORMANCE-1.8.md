### 6-b/c：NoteIndex 搜索 ✅ M4 回退后完全恢复

| 查询类型 | v1.6.5 最优 | v1.8.5（M4 逆效应）| **v1.8.6（M4 回退）** | vs v1.6.5 |
|---------|------------|-------------------|----------------------|-----------|
| ASCII | 192 µs | 1855 µs ⚠ | **187 µs** | **-3% ✅** |
| CJK | 618 µs | 2015 µs ⚠ | **531 µs** | **-14% ✅** |

**M4 回退说明（v1.8.6）**：

v1.7.0 引入的 M4 评分 Bitset 方案在大规模笔记库中产生逆效应。

```
M4 方案（已回退）：
  对每个候选笔记，遍历 content_tokens 集合建 u8 位掩码
  = O(n_content_tokens) per note  → 内容词汇量大时极慢

v1.8.6 恢复方案：
  对 ≤8 个 query_tokens 逐一查 title/tag/content 的 HashSet
  = O(8 × 3) = O(24 次 O(1) 查询) per note  → 不随内容规模变化
```

CJK 搜索 -14%（比 v1.6.5 更快）：依赖升级 + Bitset 候选集合并（M3，保留）的联合效果。
